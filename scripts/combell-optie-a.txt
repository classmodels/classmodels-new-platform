COMBELL OPTIE A — één Node-app voor www + api (classmodels-web)
================================================================================

WAT JIJ IN HET COMBELL-PANEEL ZET (Node.js → classmodels-web → Environment variables)
--------------------------------------------------------------------------------
COMBELL_HOST_ROUTER=1
# ↑ exact cijfer 1. Waarden zoals "." of leeg → dual-proxy UIT → alleen Next, geen Nest → geen API/foto’s.

DB_URL=... (zie hieronder: volledige MySQL-URL van Combell)
JWT_SECRET=... (lang willekeurig geheim)
CORS_ORIGIN=https://www.class-models.be
API_PUBLIC_URL=https://api.class-models.be
NEXT_PUBLIC_API_URL=https://api.class-models.be
NEXT_PUBLIC_APP_URL=https://www.class-models.be
WEB_PUBLIC_URL=https://www.class-models.be
WEB_APP_URL=https://www.class-models.be
NODE_ENV=production
# Mediatheek: blijft staan bij elke pipeline-deploy. Wordt automatisch gezet door combell-dual-proxy
# vóór de API start (absoluut pad, typisch $HOME/www/cm-media/uploads). Zet zelf een absoluut pad
# als je géén dual-proxy gebruikt. Vermijd MEDIA_ROOT=uploads in apps/api/.env — dat overschrijft
# de persistente map en schrijft terug naar de release-map (bestanden weg na deploy).
# MEDIA_ROOT=/home/JOUWUSER/www/cm-media/uploads

Poort in Combell (veld "Poort"): meestal 3000 — NIET wijzigen tenzij Combell anders zegt.

Website-backend (Websites & SSL → Beheer website):
  www.class-models.be  → Node → classmodels-web
  api.class-models.be → Node → classmodels-web  (zelfde app)

Pipeline (Combell → Node.js-app → tab Build / Pipeline):
  AANBEVOLEN — géén Docker/CaaS-containerbuild, alleen scripts:
    Build command:  npm run combell:build
    Start command:  npm run serve

  FOUT in build-log? Zie je een ENORME lijst node-* pakketten en
  "RUN apt -y update && apt -y upgrade && apt install npm yarn"?
  → Combell gebruikt dan hun STANDAARD CaasDockerfile (±998 bytes), NIET onze repo.
  → Zet in het paneel expliciet build via bovenstaand npm-script, of koppel Git-repo
    zodat CaasDockerfile-nodeapp uit de repo wordt gebruikt (onze: alleen git +
    ca-certificates, daarna npm ci + combell:build — geen apt upgrade).

  Docker-build faalt soms met "rpc error ... EOF" tijdens apt upgrade in Combell's
  standaard template. Gebruik npm run combell:build i.p.v. zware apt-stap.

  Daarna: app herstarten (of wacht op automatische restart). Geen aparte SSH-stappen voor
  shared/API-build: het script bouwt @cm/shared → @cm/api → @cm/web in de juiste volgorde.
  Migraties: bij start van de app (dual-proxy) draait prisma migrate deploy als DB_URL gezet is.
  De Nest-API laadt agenda-mailhulp **lokaal** (apps/api/src/agenda), niet via `node_modules/@cm/shared`,
  zodat een kapotte workspace-link op de server geen 500 meer geeft op login/catalogus.

  Let op: omgevingsvariabelen voor **COMBELL_HOST_ROUTER**, **DB_URL**, enz. moeten bij de **draaiende
  Node-app** (Combell → Node.js → classmodels-web → Environment) staan — niet alleen in een externe
  CI “pipeline” als die de runtime van de app niet overschrijft. Anders blijft COMBELL_HOST_ROUTER
  leeg of fout en start geen dual-proxy.
